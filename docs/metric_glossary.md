# Metric Glossary

Alphabetical reference of all terms, acronyms, and metric names used in this project.

---

| Term | Definition |
|---|---|
| **AOV** | Average Order Value — net revenue divided by number of transactions. |
| **At-Risk** | RFM segment: customers who purchased frequently and spent heavily but have not ordered recently (low R, high F and M). Candidates for win-back campaigns. |
| **CAGR** | Compound Annual Growth Rate — `(Ending Value / Beginning Value)^(1/n) − 1`, where n = number of years. |
| **Champions** | RFM segment: customers with high recency, high frequency, and high monetary value (R ≥ 4, F ≥ 4, M ≥ 4). Highest-priority retention targets. |
| **Cohort** | A group of customers who signed up in the same month, used for retention analysis. |
| **COGS** | Cost of Goods Sold — the direct cost of producing/acquiring the products sold. In this dataset, `unit_cost × quantity`. |
| **CTAS** | CREATE TABLE AS SELECT — an Athena DDL command that materializes query results as a new table in S3/Glue. |
| **Customer LTV** | Customer Lifetime Value — total historical net revenue generated by a customer. |
| **Decile** | One of ten equal-sized buckets when customers or products are ranked by a metric. Top decile = highest performers. |
| **Discount** | The fractional price reduction applied to a transaction (e.g., `0.15` = 15% off). |
| **DPU** | Data Processing Unit — the unit of compute used by AWS Glue (1 DPU = 4 vCPUs, 16 GB RAM). |
| **Frequency (F)** | In RFM: the total number of distinct transactions a customer has made. |
| **Glue Catalog** | AWS Glue Data Catalog — a managed metadata repository that stores table schemas, partition info, and database definitions. Athena uses it as its schema registry. |
| **Gross Margin %** | `(Net Revenue − COGS) / Net Revenue × 100`. Measures how efficiently products are sold relative to their cost. |
| **Gross Profit** | Net Revenue minus COGS. Does not include operating expenses. |
| **Hive Partitioning** | A directory naming convention (`key=value/`) used to partition data in S3 so query engines can skip irrelevant files. |
| **LPAD** | SQL string function to left-pad a string with a specified character (e.g., `LPAD('3', 2, '0')` → `'03'`). Used to build date strings. |
| **LTV** | See *Customer LTV*. |
| **Market Share %** | A region's revenue as a percentage of total revenue across all regions. |
| **Medallion Pattern** | A data lake architecture with three quality layers: raw (bronze), processed (silver), and curated/aggregated (gold). |
| **MoM** | Month-over-Month — a comparison of a metric between one month and the immediately preceding month. |
| **Monetary (M)** | In RFM: the total net revenue a customer has spent across all transactions. |
| **MSCK REPAIR TABLE** | Athena command to scan an S3 location and register all Hive-partition directories into the Glue Catalog. |
| **Net Revenue** | `unit_price × quantity × (1 − discount)`. The actual amount collected after discounts. Excludes taxes and returns. |
| **NTILE(n)** | SQL window function that divides rows into n equal buckets. `NTILE(5)` creates quintiles (1–5). |
| **Orphan Record** | A row in a child table whose foreign key does not match any record in the parent table — a referential integrity violation. |
| **Pareto Principle** | The observation that roughly 80% of outcomes come from 20% of causes — often seen as "top 20% of customers generate ~80% of revenue." |
| **Partition Pruning** | The query engine's ability to skip S3 file directories that don't match a filter predicate, reducing data scanned and cost. |
| **Parquet** | A columnar, open-source file format optimized for analytical queries. Queries only read the columns they need. |
| **Percentile Rank** | A window function (`PERCENT_RANK`) that expresses a row's relative position in a result set as a value between 0 and 1. |
| **QoQ** | Quarter-over-Quarter — a comparison between consecutive quarters. |
| **Recency (R)** | In RFM: the number of days since a customer's most recent purchase (lower = better). |
| **Retention Rate** | The percentage of a signup cohort that made at least one purchase in a given subsequent month. |
| **RFM** | Recency, Frequency, Monetary — a classic customer segmentation framework. Each dimension is scored 1–5; higher is better. |
| **RLS** | Row-Level Security — a QuickSight feature that restricts which rows of data each user or group can see. |
| **Running Total** | A cumulative sum computed using a window function `SUM(...) OVER (ORDER BY ... ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW)`. |
| **SPICE** | Super-fast, Parallel, In-memory Calculation Engine — QuickSight's managed in-memory data store. SPICE enables sub-second dashboard rendering without querying Athena on every page load. |
| **SSE-S3** | Server-Side Encryption with Amazon S3-managed keys — default encryption for S3 objects. |
| **Sub-Category** | A finer classification within a product Category (e.g., `Technology > Phones`). |
| **Unit Cost** | The cost of goods sold per individual unit, sourced from the `products` table. |
| **Unit Price** | The per-unit price agreed at the time of sale, stored in `sales_transactions`. May differ from the catalog `products.unit_price` due to promotions. |
| **Win-back Campaign** | A targeted marketing effort aimed at re-engaging At-Risk or Lost customers before they permanently churn. |
| **YoY** | Year-over-Year — a comparison of a metric between the current period and the same period in the prior year. |
| **YTD** | Year-to-Date — cumulative metric from January 1 of the current year to the current date. |
| **Z-score** | A statistical measure of how many standard deviations a value is from the mean. Used in DQ-07 to flag outlier transactions. |
